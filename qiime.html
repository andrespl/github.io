Ejecutar y correr pipeline de la imagen de QIIME

Para ejecutar QIIME primero hay que activar el entorno, para ello, en la línea de comandos del terminal debe escribirse: source activate qiime1. Una vez activo, puede manejarse igual que se manejarse como se manejaría el terminal. Para realizar el análisis que se propone en el tutorial, los pasos a seguir son:
  1. Validar el archivo de mapeo. Este archivo contiene toda la información necesaria para llevar a cabo el análisis, por ello es fundamental asegurarse de que tenga el formato correcto. Utilizando validate_mapping_file.py -m Fasting_Map.txt -o mapping_output se consigue. Si aparece algún error este se escribirá en un fichero html.
  2. El siguiente paso es asignar lecturas multiplexed a muestras basándose en el “código de barras”. Este paso además realiza un filtro de calidad basado en las características de cada secuencia, eliminando cualquiera de baja calidad o lecturas ambiguas. Se utiliza el siguiente código: split_libraries.py -m Fasting_Map.txt -f Fasting_Example.fna -q Fasting_Example.qual -o split_library_output.
  3. Utilizando uno de los output del paso anterior, seqs.fna, se ejecuta la línea: pick_de_novo_otus.py -i split_library_output/seqs.fna -o otus.
  4. Un sumario de los contenidos de la tabla OTU pueden observarse usando el comando: biom summarize-table -i otus/otu_table.biom
